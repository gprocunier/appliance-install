Tutorial

Full Lab First Run

Prepare the virtualization host, create foundry, build the appliance image, boot three VMs, and verify the cluster.

Start Here GitHub Repo

Outcome

Three-node demo

The end state is an installed OpenShift 4.21 appliance cluster on ocp-01, ocp-02, and ocp-03.

Use this tutorial when the lab is new or when you are rehearsing the workshop from the beginning. Run every command from the repository root on the operator workstation.

1. Create Local Config

The tracked examples are safe to publish. The copied files are ignored by git and hold the real lab values.

cp config/host.env.example config/host.env
cp config/rhsm.env.example config/rhsm.env
cp config/network.env.example config/network.env
cp config/foundry.env.example config/foundry.env
cp config/appliance.env.example config/appliance.env
cp config/operators.env.example config/operators.env
cp config/additional-images.env.example config/additional-images.env

Edit the copied files before continuing. Replace placeholders for host access, Red Hat registration, pull secret path, console passwords, DNS forwarders, and image locations.

2. Prepare The Virtualization Host

Scripts 01 through 05 register the remote host, install packages, enable services, create OVS/libvirt networking, and verify the baseline.

./scripts/01-register-rhn.sh
./scripts/02-install-host-packages.sh
# wait for the virtualization host to reboot
./scripts/03-enable-host-services.sh
./scripts/04-configure-ovs-networks.sh
./scripts/05-verify-virt-host.sh

Script 02 intentionally reboots the host. Wait until SSH returns before running script 03.

3. Create And Configure Foundry

Foundry is dual-homed: one NIC reaches upstream content and one NIC serves the isolated appliance network.

./scripts/06-create-foundry-vm.sh
./scripts/07-configure-foundry-console.sh
./scripts/08-configure-foundry-services.sh
./scripts/09-verify-foundry-services.sh

Script 08 can be long while packages install and IdM configures DNS. Script 09 is the checkpoint for DNS, NTP, HTTP staging, and firewall state.

4. Build Appliance Assets

Scripts 10 through 12 prepare appliance inputs on foundry, build appliance.raw, and create the Agent Installer config ISO.

./scripts/10-prepare-appliance-assets.sh
./scripts/11-build-appliance-image.sh
./scripts/12-create-cluster-config-image.sh

The build step mirrors OpenShift release content, configured operators, and optional additional images. Expect this phase to take the longest.

5. Boot And Verify The Cluster

Script 13 pulls the artifacts from foundry to the virtualization host, converts the base image when needed, and creates the three OpenShift VMs.

./scripts/13-create-ocp-vms.sh
./scripts/15-watch-ocp-install.sh
./scripts/16-verify-ocp-cluster.sh

Script 14 is deliberately skipped on the first run. It is only for removing existing node VMs before a reimage.

Next Step

After the first successful install, use Reimage The Cluster to reset the demo quickly without rebuilding the entire environment.
