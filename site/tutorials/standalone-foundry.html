Tutorial

Standalone Foundry Build

Mirror content and produce appliance.raw on an existing RHEL 10.x server with normal flat networking.

Start Here GitHub Repo

Scope

Build only

This path does not create VMs, OVS networks, IdM DNS, NTP service, HTTP staging, or a cluster config ISO.

Use the standalone path when a RHEL 10.x server can reach the internet through its default gateway and only needs to build the appliance disk image.

1. Create Local Config

cp config/foundry-standalone.env.example config/foundry-standalone.env
cp config/rhsm.env.example config/rhsm.env
cp config/appliance.env.example config/appliance.env
cp config/operators.env.example config/operators.env
cp config/additional-images.env.example config/additional-images.env

Set APPLIANCE_STANDALONE_HOST, APPLIANCE_STANDALONE_USER, APPLIANCE_STANDALONE_SSH_KEY, and APPLIANCE_PULL_SECRET_FILE in the ignored local files.

2. Register And Verify The Host

./scripts/foundry-standalone/01-register-rhn.sh
./scripts/foundry-standalone/02-install-packages.sh
./scripts/foundry-standalone/03-verify-host.sh

The verification script checks RHEL 10.x, default route, DNS, registry reachability, and required build tools.

3. Prepare And Build

./scripts/foundry-standalone/04-prepare-appliance-assets.sh
./scripts/foundry-standalone/05-build-appliance-image.sh

The output image is written on the standalone host at ${APPLIANCE_ASSETS_DIR}/appliance.raw. With tracked defaults, that is /srv/appliance/assets/appliance.raw.

4. Optionally Fetch The Image

./scripts/foundry-standalone/06-fetch-appliance-image.sh

The optional fetch script copies appliance.raw into APPLIANCE_STANDALONE_LOCAL_OUTPUT_DIR, which defaults to work/foundry-standalone.

Use the full lab tutorial when the workshop also needs VM deployment and disconnected network services.
