Architecture

Why the lab separates operator control, virtualization, foundry services, and disconnected OpenShift nodes.

Core Idea

Foundry is the boundary

Foundry can prepare content upstream, then serve the isolated appliance network without making the OpenShift nodes internet-routable.

The lab is designed to be understandable during a customer demonstration: every step is run from the operator workstation, but each system has a narrow responsibility.

Disconnected Network Model

The three OpenShift VMs are single-homed on lab-switch. The OVS bridge has no physical uplink and the host does not provide NAT for the appliance network.

DNS And NTP

Foundry runs IdM-backed DNS for appliance.workshop.lan and NTP for the appliance network. Script 10 writes the configured NTP source into agent-config.yaml as additionalNTPSources.

IdM forwarders are configured so foundry itself can resolve Red Hat CDN names while preparing content. Recursion and query-cache access are restricted to localhost, so the appliance network gets lab DNS without becoming an internet resolver path.
