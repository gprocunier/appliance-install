How-to: Customize Mirrored Content. Change what gets baked into appliance.raw without putting secrets in tracked files.

Start Here GitHub Repo

Rule

Ignored files only

Real pull secrets, registry tokens, private image refs, and environment-specific choices belong in ignored local config.

Use this guide when the workshop needs a different operator set, IBM Cloud Pak content, private Quay content, or other non-operator image refs in the appliance image.

Change Operator Packages

Edit the ignored operator config. Script 10 reads it and writes the ApplianceConfig operator section.

cp config/operators.env.example config/operators.env
vi config/operators.env

The default list includes OpenShift Virtualization, ODF, NMState, cert-manager, Network Observability, Web Terminal, and Quay.

Add Non-Operator Images

Use config/additional-images.env for a short list, or point it at an ignored one-image-per-line file for larger content sets.

cp config/additional-images.env.example config/additional-images.env
cp config/cloudpak.images.example config/cloudpak.images
vi config/additional-images.env
vi config/cloudpak.images

Add Registry Auth

The pull secret referenced by APPLIANCE_PULL_SECRET_FILE must include every private registry used by operator catalogs or additional images.

cp config/pull-secret.multi-registry.json.example \
    config/pull-secret.multi-registry.json

podman login --authfile config/pull-secret.multi-registry.json registry.redhat.io
podman login --authfile config/pull-secret.multi-registry.json quay.io
podman login --authfile config/pull-secret.multi-registry.json cp.icr.io

Point ignored config/appliance.env at the real pull secret file with an absolute path.

Rebuild After Content Changes

./scripts/10-prepare-appliance-assets.sh
./scripts/11-build-appliance-image.sh
./scripts/12-create-cluster-config-image.sh
./scripts/14-destroy-ocp-vms.sh
APPLIANCE_REFRESH_BASE_IMAGE=true ./scripts/13-create-ocp-vms.sh

Changing mirrored content changes appliance.raw, not only the Agent Installer config ISO. Existing OpenShift VMs must be removed before script 13 can recreate them from the refreshed base image.

Script 10 validates pull-secret JSON and rejects placeholder auth values before copying it to foundry.
