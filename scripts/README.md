# Scripts

Scripts in this directory should be numbered in the order an operator runs
them. Keep each script focused on one setup phase.

Run numbered scripts from the repository root. The scripts load local
configuration from `config/*.env` and use `scripts/lib/remote.sh` to connect to
the target virtualization host over SSH. Scripts that operate inside foundry use
the virtualization host as an SSH jump path.

`scripts/lib/remote.sh` is shared helper code, not a script to run directly.
See `docs/execution-model.md` for the full execution model.

First build golden path:

```bash
./scripts/01-register-rhn.sh
./scripts/02-install-host-packages.sh
# wait for the virtualization host to reboot
./scripts/03-enable-host-services.sh
./scripts/04-configure-ovs-networks.sh
./scripts/05-verify-virt-host.sh
./scripts/06-create-foundry-vm.sh
./scripts/07-configure-foundry-console.sh
./scripts/08-configure-foundry-services.sh
./scripts/09-verify-foundry-services.sh
./scripts/10-prepare-appliance-assets.sh
./scripts/11-build-appliance-image.sh
./scripts/12-create-cluster-config-image.sh
./scripts/13-create-ocp-vms.sh
./scripts/15-watch-ocp-install.sh
./scripts/16-verify-ocp-cluster.sh
```

Script `14` is reserved for reimage cleanup. Do not run it between first VM
creation and install watch.

Quick reimage path:

```bash
# Optional when cluster config, NTP, networking, nodes, or pull-secret inputs changed:
./scripts/10-prepare-appliance-assets.sh
./scripts/12-create-cluster-config-image.sh

./scripts/14-destroy-ocp-vms.sh
./scripts/13-create-ocp-vms.sh
./scripts/15-watch-ocp-install.sh
./scripts/16-verify-ocp-cluster.sh
```

Use `./scripts/14-destroy-ocp-vms.sh` only when you need to remove the current
OpenShift VMs and reimage them with script `13`. Script `13` reuses an existing
`appliance-base.qcow2` by default when `APPLIANCE_REFRESH_BASE_IMAGE=false`, so
the quick reimage path normally refreshes only `agentconfig.noarch.iso` and the
node overlay disks. Script `12` regenerates installer state for the next install,
including the foundry-local kubeconfig.

When `config/operators.env` or `config/additional-images.env` changes, the
mirrored content in `appliance.raw` must change too. Rerun script `10`, rerun
script `11`, then run script `13` with `APPLIANCE_REFRESH_BASE_IMAGE=true` so
the OpenShift VMs boot from the rebuilt base image.

Script groups:

| Scripts | Purpose |
| --- | --- |
| `01` through `05` | Prepare and verify the virtualization host. |
| `06` through `09` | Create, configure, and verify foundry. |
| `10` through `12` | Prepare OpenShift appliance assets, build `appliance.raw`, and create the Agent Installer config ISO on foundry. |
| `13` through `16` | Create or reimage `ocp-01`, `ocp-02`, and `ocp-03`, watch the install, then verify the cluster. |

First-pass operator flow:

| Step | Run | Expected output pattern |
| --- | --- | --- |
| Prepare local config | Copy each `config/*.env.example` to the matching ignored `config/*.env` file. | No secrets are printed or committed; placeholders are replaced only in ignored local files. |
| Host setup | `01`, `02`, wait for reboot, then `03`, `04`, `05` | Package and service commands stream output. Script `02` ends the SSH session when the host reboots. |
| Foundry setup | `06`, `07`, `08`, `09` | Script `06` prints wait attempts for foundry SSH. Script `08` may be quiet during `dnf install` and IdM setup. Script `09` is the read-only checkpoint. |
| OpenShift 4.21 assets | `10`, `11`, `12` | Script `10` writes generated YAML on foundry, including `additionalNTPSources` in `agent-config.yaml` from `APPLIANCE_AGENT_NTP_SOURCE`, operator packages from `config/operators.env`, and optional image refs from `config/additional-images.env`. Script `11` can be quiet while mirroring release, operator, and additional image content, then prints `qemu-img info` and `ls -lh` for `appliance.raw`. Script `12` prints the `openshift-install` version and the finished ISO path. |
| OpenShift VMs | `13`, then `15`, then `16` | Script `13` uses a temporary host-generated SSH key to pull sparse artifacts directly from foundry to the virtualization host, avoiding workstation hairpin. Script `15` waits for bootstrap and install completion. Script `16` verifies the installed cluster through a temporary local API tunnel. |

The default appliance package list generated by script `10` covers OpenShift
Virtualization, ODF, NMState, cert-manager, Network Observability, Web
Terminal, and Quay. Customize it in ignored `config/operators.env`. Do not paste
pull-secret content or generated `appliance-config.yaml` contents into tracked
documentation.

Use ignored `config/additional-images.env` for IBM Cloud Pak or other private
registry image refs that must be mirrored as ApplianceConfig
`additionalImages`. Registry credentials for those images belong in the ignored
pull secret file referenced by `APPLIANCE_PULL_SECRET_FILE`. Large image lists
can live in ignored `config/*.images` files referenced by
`APPLIANCE_ADDITIONAL_IMAGES_FILE`. See `docs/registry-auth.md` for copyable
examples that combine Red Hat, IBM, Quay, and generic private registry auth.

Style rules:

- Use names like `01-prepare-host.sh`, `02-create-foundry.sh`, and
  `03-create-ocp-vms.sh`.
- Avoid clever pipe-heavy one-liners.
- Prefer readable, commented commands.
- Use `####` for high-level sections.
- Use `#` for important individual steps.
